# Коллекция Ansible VKR.ans_vkr_skv

Коллекция предназначена для Демонстрации по автоматизации развертывания и настройки гибридной сетевой инфраструктуры на базе Linux в составе домена Active Directory с использованием Samba, прокси-сервера Squid с Kerberos-аутентификацией, сервисов DHCP с failover и динамическим обновлением DNS, а также файловых серверов SMB и NFS с интеграцией в домен.

## Назначение

Данная коллекция разработана в рамках выпускной квалификационной работы по теме Проектирование и автоматизация внедрения гибридной сетевой инфраструктуры на базе Ansible в составе домена AD, прокси-сервера SQUID и Динамического DNS.

Коллекция обеспечивает полное автоматизированное развертывание инфраструктуры, включая:

- Подготовку управляющего и управляемых узлов
- Базовую настройку хостов (обновление, настройка сети, отключение IPv6, синхронизация времени)
- Развертывание контроллеров домена Samba Active Directory с репликацией
- Настройку DHCP-сервера с поддержкой failover и динамическим обновлением DNS
- Репликацию SysVol между контроллерами домена с использованием Unison и rsync
- Развертывание файловых серверов SMB и NFS с интеграцией в домен и настройкой прав доступа
- Настройку прокси-сервера Squid с аутентификацией через Kerberos и контролем доступа по группам домена

## Структура коллекции

Коллекция включает следующие роли:

- base_setup - базовая настройка хостов: обновление пакетов, настройка имени хоста, резолвера, отключение IPv6
- chrony_sync - синхронизация времени через Chrony с иерархией серверов для контроллеров домена и клиентов
- samba_ad_dc - развертывание контроллеров домена Samba Active Directory, настройка DNS, Kerberos, репликация
- dhcp_server - настройка DHCP-сервера с поддержкой failover и скриптом динамического обновления DNS через samba-tool
- sysvol_replication - двунаправленная репликация каталога SysVol между контроллерами домена через Unison/rsync
- smb_shares - настройка файлового сервера SMB с доменной аутентификацией, создание общих ресурсов и управление правами
- nfs_server - настройка файлового сервера NFS с поддержкой Kerberos-аутентификации (SECURE_NFS)
- squid_proxy - развертывание прокси-сервера Squid с аутентификацией через Kerberos и контролем доступа по группам домена

## Требования

- Управляющий узел: Linux с установленным Ansible 2.9+ или ansible-core 2.11+
- Управляемые узлы: совместимые дистрибутивы Linux (ALT Linux, CentOS, Rocky Linux, Ubuntu)
- Доступ к управляемым узлам по SSH с использованием ключей
- Учетная запись с правами sudo или su для выполнения задач с повышенными привилегиями
- Для работы с доменом: сетевая связность между узлами, настроенные обратные зоны DNS, внешний NTP-сервер

## Использование

### Подготовка инвентаря

Создайте файл инвентаря inventory/inventory.yaml с описанием групп хостов:

```yaml
all:
  children:
    domain_controllers:
      hosts:
        altsrv2:
          ansible_host: 192.168.100.12
        altsrv3:
          ansible_host: 192.168.100.13
    file_servers:
      hosts:
        altsrv4:
          ansible_host: 192.168.100.14
    proxy_servers:
      hosts:
        altsrv1:
          ansible_host: 192.168.100.11
```

### Настройка переменных

Глобальные переменные определяются в inventory/group_vars/all/all.yml, чувствительные данные (пароли, ключи) хранятся в зашифрованном файле vault с использованием Ansible Vault.

Пример базовых переменных:

```yaml
ad_workgroup: "den.skv"
ad_realm: "DEN.SKV"
ad_domain: "DEN"
ad_admin_user: "Administrator"
dns_forwarder: "77.88.8.8"
```

### Запуск плейбука

Основной плейбук main.yaml импортирует отдельные плейбуки для каждой роли с возможностью выборочного включения через переменные:

```bash
ansible-playbook -i inventory/inventory.yaml main.yaml --vault-password-file ./va_pa
```

Для запуска конкретной роли:

```bash
ansible-playbook -i inventory/inventory.yaml samba_ad_dc.yaml --vault-password-file ./va_pa
```

## Описание ролей

### base_setup

Выполняет первоначальную настройку всех хостов инфраструктуры:
- Обновление кеша пакетов и установленных приложений
- Обновление ядра системы
- Установка имени хоста в формате имя.домен
- Определение основного сетевого интерфейса и настройка DNS-резолвера
- Отключение IPv6 через sysctl с созданием systemd-службы для применения настроек после загрузки
- Перезагрузка хоста после завершения настроек

### chrony_sync

Настраивает синхронизацию времени через Chrony:
- Основной контроллер домена получает время от внешнего NTP-сервера и раздает его вторичному контроллеру и клиентам
- Вторичный контроллер синхронизируется с основным и внешним источником
- Клиенты домена получают время только от контроллеров домена
- Автоматический перезапуск службы chronyd при изменении конфигурации

### samba_ad_dc

Развертывает контроллеры домена Samba Active Directory:
- Остановка и маскирование конфликтующих служб (smb, nmb, bind, slapd)
- Установка необходимых пакетов task-samba-dc
- Provisioning основного контроллера с настройкой параметров домена, DNS-бэкенда, политик обновлений
- Присоединение вторичного контроллера с репликацией данных
- Создание обратных зон DNS и записей PTR для контроллеров
- Настройка резолвера и конфигурации Kerberos для клиентского доступа
- Обработчики для перезапуска сетевых служб без потери соединения

### dhcp_server

Настраивает DHCP-сервер с расширенными возможностями:
- Создание служебного пользователя dhcpduser в домене для динамического обновления DNS
- Экспорт keytab-файла для аутентификации службы
- Развертывание скрипта dhcp-dyndns.sh для безопасного обновления записей A и PTR через samba-tool
- Поддержка конфигурации с failover для двух контроллеров домена с синхронизацией состояния пулов адресов
- Настройка параметров аренды, диапазона адресов, опций домена и серверов

### sysvol_replication

Обеспечивает репликацию каталога SysVol между контроллерами домена:
- Генерация и обмен SSH-ключами для безопасного доступа между контроллерами
- Настройка конфигурации Unison для двунаправленной синхронизации
- Первоначальная синхронизация через rsync с сохранением расширенных атрибутов
- Создание systemd timer для периодической синхронизации каждые 5 минут
- Логирование процесса репликации в отдельный файл

### smb_shares

Развертывает файловый сервер SMB с доменной интеграцией:
- Регистрация записей DNS для сервера файлов
- Создание доменных групп и пользователей с заданными параметрами
- Присоединение сервера к домену через system-auth
- Настройка smb.conf с параметрами безопасности ADS и интеграции с SSSD
- Создание общих ресурсов с гибкой настройкой прав доступа через списки read/write
- Настройка масок создания файлов и каталогов, управление видимостью ресурсов

### nfs_server

Настраивает файловый сервер NFS с поддержкой Kerberos:
- Включение режима SECURE_NFS для аутентификации через Kerberos
- Создание служебного принципала nfs и добавление его в keytab
- Настройка экспорта ресурсов с опциями sec=krb5:krb5i:krb5p
- Интеграция с доменом для разрешения имен пользователей и групп
- Управление правами доступа на уровне каталогов с учетом доменных групп

### squid_proxy

Развертывает прокси-сервер Squid с аутентификацией через домен:
- Создание группы доступа proxy_acc и добавление в нее разрешенных пользователей
- Получение принципала HTTP для аутентификации прокси-сервера в домене
- Настройка negotiate_kerberos_auth для прозрачной аутентификации пользователей
- Интеграция с ext_kerberos_ldap_group_acl для контроля доступа по группам домена
- Настройка параметров кэширования, ограничений по размеру объектов и портов

## Переменные

Все роли поддерживают переопределение параметров через переменные. Основные группы переменных:

Общие параметры:
- ad_workgroup, ad_realm, ad_domain - параметры домена
- ad_admin_user, ad_admin_password - учетные данные администратора
- dns_forwarder - внешний DNS-форвардер
- первичный и вторичный контроллеры домена через groups и hostvars

Управление включением ролей:
- Переменные вида role_name: true/false для выборочного запуска компонентов

Параметры ролей:
- chrony_sync: exter_ntp, allow_clients
- samba_ad_dc: ad_backend, dns_refresh, ptr_zone
- dhcp_server: network_subnet, dhcp_range, lease_time
- smb_shares: smb_shares_config с параметрами каждого ресурса
- squid_proxy: cache_mem, cache_dir, negotiate_param

Чувствительные данные хранятся в зашифрованном файле vault:
- ad_admin_password, ansible_become_password, vault_omapi_secret
